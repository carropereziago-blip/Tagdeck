# Roadmap

This roadmap is directional. It may change as testers use TagDeck with real AI
music libraries.

## 1. Tester-Ready Desktop Build

- Stable Windows desktop build.
- Safe scanning and playback.
- Library performance for large folders.
- Clear onboarding.
- Strong `.gitignore` and public repository hygiene.

## 2. Workflow Presets

- Useful review flows instead of simple field presets.
- Fast decisions for radio, release, DAW finishing, archive cleanup and model
  seed collection.
- Better keyboard-first review.

## 3. Performance

- Faster library search.
- Better visible limits and virtualized tables.
- Safer handling of very large libraries.
- Faster export previews.

## 4. Export Packs

- DAW prep packs.
- Radio packs.
- Release packs.
- Model-seed packs.
- Archive packs.

The first versions should copy files only. They should not move originals.

## 5. Mobile / Sync Readiness

- Stable IDs.
- Relative paths.
- Manifest export.
- Future mobile change import.
- Explicit conflict handling.

See [MOBILE_SYNC_READINESS.md](MOBILE_SYNC_READINESS.md).

## 6. Track Workspace

- Workspace around a song or song family.
- Related versions, remasters, stems, lyrics, prompts, cover art and DAW notes.
- Future package/export history.

See [TRACK_WORKSPACE_ROADMAP.md](TRACK_WORKSPACE_ROADMAP.md).

## 7. Companion Mobile App Later

The mobile app is a future companion, not the source of truth.

```text
Desktop owns the library.
Mobile reviews the library.
Sync merges the decisions.
```
